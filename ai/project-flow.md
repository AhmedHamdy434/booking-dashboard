# ملخص المشروع: تطبيق حجز صالون الحلاقة (Barbershop Booking App)

هذا الملف يمثل المرجع الشامل للمشروع، ويحتوي على تفاصيل المنطق البرمجي، وهيكلية البيانات، وكيفية الإدارة.

## 1. فلو التطبيق (App Flow)
التطبيق مقسم إلى 3 خطوات أساسية مع مؤشر تقدم (Progress Bar):
- **الخدمة (Service)**: اختيار الخدمة المطلوبة (معرفة السعر والمدة).
- **الحلاق (Barber)**: اختيار الحلاق المفضل.
- **الموعد (Schedule)**: اختيار اليوم والوقت (توليد المواعيد يتم بناءً على مدة الخدمة).

## 2. المنطق البرمجي للحجز (Booking Logic)
- **توليد المواعيد (Slot Generation)**: 
    - يتم تقسيم يوم العمل إلى فترات (Intervals) مدتها 15 دقيقة.
    - يتم التحقق من كل فترة: "هل إذا بدأ الحجز الآن، سينتهي قبل بداية الحجز التالي؟".
- **حماية البيانات**:
    - التطبيق يمنع الحجز المتعارض من الواجهة.
    - قاعدة البيانات (Supabase) تحتوي على `Trigger` يمنع أي حجز متعارض برمجياً كطبقة حماية ثانية.

## 3. حالات الحجز (Booking Statuses)
| الحالة | الوصف | اللون في التطبيق |
| :--- | :--- | :--- |
| **Pending** | حجز جديد بانتظار موافقة الأدمن | 🟡 أصفر |
| **Confirmed** | تمت الموافقة عليه من قبل الأدمن | 🟢 أخضر |
| **Completed** | انتهت الخدمة بنجاح | 🔵 أزرق |
| **Cancelled** | تم رفض الحجز أو إلغاؤه | 🔴 أحمر |

## 4. لوحة تحكم الأدمن (Admin Dashboard)
الأدمن هو المسؤول عن:
- مراجعة الحجوزات المعلقة (`Pending`) والموافقة عليها.
- إدارة الخدمات (إضافة/تعديل/حذف).
- إدارة الحلاقين وساعات عملهم.

## 5. تعليمات التشغيل النهائية
لضمان عمل النظام بكفاءة، يجب التأكد من:
1. **قواعد البيانات**: تطبيق كود SQL الموجود في ملف `supabase_setup.sql`.
2. **البيئة**: التأكد من وجود مفاتيح Supabase الصحيحة في ملف `.env`.
3. **التشغيل**: استخدام `npm start` لتشغيل تطبيق Expo.
